# Въведение в Бази Данни и SQL

Тази лекция ще ви запознае с основните концепции за управление на данни, какво е SQL и как да създавате първите си таблици.

## 1. Управление на данни (Data Management)

Управлението на данни включва събирането, съхранението и използването на данни по сигурен и ефективен начин. В програмирането, когато говорим за съхранение на информация, обикновено използваме **Бази от данни (Databases)**.

Базата данни е организирана колекция от информация, която лесно може да бъде достъпвана, управлявана и обновявана.

## 2. SQL и Structured Query Language

**SQL** (произнася се "Ес-Кю-Ел" или "Сикуел") означава **Structured Query Language** (Език за структурирани заявки).

- Това е стандартният език за работа с релационни бази данни.
- Чрез SQL можем да:
  - Създаваме нови бази данни и таблици.
  - Вмъкваме (Insert) данни.
  - Търсим (Select) данни.
  - Променяме (Update) данни.
  - Изтриваме (Delete) данни.

## 3. Типове данни (Data Types)

Когато създаваме таблица, трябва да укажем какъв вид информация ще съхранява всяка колона. Ето някои от най-често използваните типове данни в SQL (например в SQLite или PostgreSQL):

- **INTEGER**: Цяло число (напр. 1, 42, -5).
- **REAL / FLOAT**: Дробно число (напр. 3.14, -0.01).
- **TEXT / VARCHAR**: Текстов низ (напр. "Иван", "София").
- **BOOLEAN**: Логическа стойност (TRUE или FALSE / 1 или 0).
- **DATE / DATETIME**: Дата и час.

## 4. Дефиниране на данни (DDL - Data Definition Language)

DDL е част от SQL, която се занимава със структурата на базата данни. Основните команди са:

### CREATE TABLE (Създаване на таблица)

Това е командата за създаване на нова таблица. Трябва да дефинираме името на таблицата и нейните колони.

**Синтаксис:**
```sql
CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype,
   ....
);
```

**Пример:**
Нека създадем таблица за ученици (`students`).

```sql
CREATE TABLE students (
    id INTEGER PRIMARY KEY,
    first_name TEXT,
    last_name TEXT,
    age INTEGER
);
```
* `PRIMARY KEY` означава, че тази колона е уникален идентификатор за всеки ред.

### DROP TABLE (Изтриване на таблица)

Тази команда изтрива цялата таблица и всички данни в нея! Внимавайте с нея.

```sql
DROP TABLE students;
```

## 5. Упражнения за начинаещи

За да изпълните тези задачи, можете да използвате онлайн SQL редактор (като [SQLFiddle](http://sqlfiddle.com/) или [SQLiteOnline](https://sqliteonline.com/)) или инсталиран софтуер (като DB Browser for SQLite).

### Задача 1: Създаване на библиотека
Създайте таблица `books`, която да съхранява информация за книги.
Таблицата трябва да има следните колони:
1. `id` - цяло число, първичен ключ (PRIMARY KEY).
2. `title` - текст (заглавие на книгата).
3. `author` - текст (автор).
4. `published_year` - цяло число (година на издаване).
5. `is_available` - булева стойност (дали книгата е налична).

<details>
<summary>Виж решението</summary>

```sql
CREATE TABLE books (
    id INTEGER PRIMARY KEY,
    title TEXT,
    author TEXT,
    published_year INTEGER,
    is_available BOOLEAN
);
```
</details>

### Задача 2: Магазин за продукти
Създайте таблица `products` със следните характеристики:
1. `product_id` - уникален номер.
2. `name` - име на продукта.
3. `price` - цена (дробно число).
4. `category` - категория (текст).

<details>
<summary>Виж решението</summary>

```sql
CREATE TABLE products (
    product_id INTEGER PRIMARY KEY,
    name TEXT,
    price REAL,
    category TEXT
);
```
</details>

### Задача 3: Добавяне на данни (INSERT - Бонус)
Въпреки че не го разгледахме подробно, ето как се добавят данни в таблицата от Задача 1:

```sql
INSERT INTO books (id, title, author, published_year, is_available)
VALUES (1, 'Под игото', 'Иван Вазов', 1894, 1);
```

Опитайте да добавите още една книга сами!
